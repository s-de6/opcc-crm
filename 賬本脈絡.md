## 銀行交易分類規則

在 `api/src/routes/file-storage.ts` 的 `importStatementFromFile()` 中定義。主要模式：

- `KONICA MINOLTA`, `XENUS TECHNOLOGY` → 41101 專業服務收入
- `CHARGES`, `手續費` → 65101 銀行手續費
- `MANULIFE` → 61202 強積金僱主供款
- `CHINA MOBILE` → 62301 電話及上網
- `SHELL` → 64301 本地交通
- `LOAN REPAYMENT` → 31201 董事往來
- `TD DESIGNATED` → 31201 董事往來
- `匯出匯款` → 51101 外判工作費用
- `MCD` → 64202 交際應酬費

未分類交易可在月結單頁面直接從下拉選單選擇科目。

## 記帳流程

1. 上傳銀行月結單 PDF → GLM-OCR + DeepSeek 解析 → 建立 bank_statements + bank_transactions
2. 在「科目」頁設定承上結餘和財政年度
3. 「記帳 → 分類帳 → 從銀行資料自動產生分錄」→ 建立 journal_entries
4. 查看試算表 / 損益表 / 資產負債表
5. 年底導出 CSV 給 auditor
