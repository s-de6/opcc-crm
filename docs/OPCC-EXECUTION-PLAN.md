## 1. 願景與定位

### 一句話

> **OPCC 是一個為香港一人公司提供「技術 + 知識 + 社群」三位一體服務的俱樂部。**

### 核心價值主張

```
技術：  一站式 CRM + AI + WhatsApp 數位基礎設施（$5/月起）
知識：  每月講座 + 工作坊 + 實戰培訓
社群：  展覽交流 + 合作配對 + 互相支援
```

### 目標客群

| 類型 | 描述 | 人數（香港估計） |
|---|---|---|
| **自由工作者** | 設計師、開發者、顧問 | ~50,000 |
| **一人公司老闆** | 已註冊公司，1 人營運 | ~30,000 |
| **斜槓創業者** | 有正職，同時經營副業 | ~100,000 |
| **退休創業** | 退休後開始一人事業 | ~10,000 |
| **數碼遊牧** | 遠端工作的獨立工作者 | ~5,000 |

### 三年目標

| 年份 | 付費會員 | 年營收（HKD） | 里程碑 |
|---|---|---|---|
| Year 1 | 50 | ~$250,000 | 平台上線 + 首次展覽 |
| Year 2 | 200 | ~$1,200,000 | 跨地區（台灣/新加坡） |
| Year 3 | 500 | ~$3,500,000 | 美國/中國版上線 |

### 4.2 多租戶架構（Multi-Tenancy）

每個會員（OPC）是獨立的租戶：

```
Cloudflare Workers (單一實例)
    ├── 租戶 A（陳先生的設計公司）
    │   ├── /api/tenants/abc123/customers
    │   ├── /api/tenants/abc123/invoices
    │   └── D1: tenant_id = abc123
    ├── 租戶 B（李小姐的顧問公司）
    │   ├── /api/tenants/def456/customers
    │   └── D1: tenant_id = def456
    └── 租戶 C（張先生的貿易公司）
        ├── /api/tenants/ghi789/customers
        └── D1: tenant_id = ghi789
```

### 9.5 盈虧平衡點

```
固定成本/月：~$4,500（Cloudflare + 域名 + 行銷 + 雜費）
變動成本/會員：~$100（AI API + 支援）

基礎會員年費：$1,200 → 月均 $100
加權平均年費：~$2,040 → 月均 $170

盈虧平衡：$4,500 ÷ ($170 - $100) ≈ 64 個付費會員
```

## 10. 風險與應對

### 10.1 技術風險

| 風險 | 機率 | 影響 | 應對策略 |
|---|---|---|---|
| WhatsApp 封號 | 低-中 | 高 | 不群發、遵守 ToS、備用 Telegram |
| Cloudflare 漲價 | 低 | 中 | Hono 可遷移到 Deno/Bun/Node |
| D1 容量不足 | 中 | 中 | 分庫 + 歷史資料歸檔到 R2 |
| DeepSeek 服務不穩 | 中 | 中 | 備用 GPT-4o mini / Claude |
| Baileys 協定失效 | 中 | 高 | 改用 QClaw 更新版 / Evolution API |

### 10.2 業務風險

| 風險 | 機率 | 影響 | 應對策略 |
|---|---|---|---|
| 會員數增長慢 | 中 | 高 | 加強行銷、提供免費體驗、口碑推薦獎勵 |
| 競爭對手出現 | 中 | 中 | 持續創新、深耕社群、建立轉換成本 |
| 講師資源不足 | 低 | 中 | 建立會員講師制度、錄影庫累積 |
| 展覽報名不足 | 中 | 中 | 與其他組織合辦、提供早鳥優惠 |

### 10.3 合規風險

| 風險 | 說明 | 應對 |
|---|---|---|
| 資料隱私 | 會員客戶資料存 D1 | 加密 + 隱私政策 + GDPR 參考 |
| WhatsApp 非官方 API | Baileys 違反 ToS | 僅供個人使用、不提供商業群發 |
| 電子支付合規 | Stripe HK 需要商業帳號 | 確保公司註冊 + BR 有效 |
| 稅務 | 服務收入需報利得稅 | 每年審計 + 準時報稅 |
