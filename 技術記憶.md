## D1 資料庫

預設 DB ID: `c455dc9a-1920-42c5-86c4-e93cec21dcfb`

所有操作需加 `--remote` 才能改生產資料庫：
```bash
cd api && npx wrangler d1 execute oppc-crm-db --remote --command "SELECT ..."
```

- `company_settings.fiscal_year_start` — 財政年度起

## Firm 模式

`api/src/middleware/auth.ts` 的 `firmContextMiddleware` 處理 `X-Active-Client` header：
- Firm admin 查詢 `firm_clients` → 取得 `client_user_id`
- 設定 `c.set('client_user_id', ...)` 
- 各 route 用 `tenantId = c.get('client_user_id') || user.id`

**關鍵 Bug：** `authMiddleware` 和 `firmContextMiddleware` 的執行順序。auth 必須先於 firm context。已在 `api/src/index.ts:63-78` 將兩者合併到 app-level middleware。

**常見 Bug 模式：** 多處 route 用 `user.id` 而非 `tenantId`。已修復的檔案：`bank-statements.ts:108`、`expense-receipts.ts:56`、`file-storage.ts:990,1004`。

## AI 助理 (chat.ts)

### 工具系統
- 50+ CRM 工具（CRUD 客戶/供應商/發票/報價單/記帳分錄/銀行交易等）
- 5 個 Claude Code 工具：`read_code`、`write_code`、`list_project_files`、`git_log`、`deploy_frontend`
- 動態 SQL 查詢：`query_database`
- Auto-categorize、import bank statement、OCR

### DSML Auto-Repair（關鍵）
當 DeepSeek 輸出 XML 格式 tool call 而非 native function call 時：
1. 正則檢測 `<invoke name="...">` 標籤
2. 解析參數、映射函數名稱
3. 執行工具 → 結果直接回傳（fast-path，不再呼叫 DeepSeek 第二次）
4. **Universal safety strip：** 所有回覆儲存前清除任何殘留 XML tag

### Streaming
- 前端送 `stream: true` → 後端包裝 reply 為 `ReadableStream (text/plain)`，不再重複呼叫 DeepSeek

### 系統提示
- 超出既有 function call 時，AI 應主動使用 code tools 實現功能
- 不允許說「我做不到」

## OCR + AI Pipeline

1. **GLM-OCR** (`api.z.ai`) — 從 PDF/圖片提取文字，支援中英文表格
2. **DeepSeek** (`api.deepseek.com`) — 解析 OCR 文字為結構化 JSON

API Keys（Worker Secrets）：
- `DEEPSEEK_API_KEY` — DeepSeek API
- `GLM_API_KEY` — Z.AI GLM-OCR  
- `GITHUB_TOKEN` — GitHub PAT（AI 讀寫程式碼用）

## Telegram 帳單導入

`api/src/routes/messaging.ts` — `processTelegramMedia()` 函數：
1. Telegram webhook 接收照片/文件
2. 下載檔案 → R2
3. GLM-OCR 辨識
4. DeepSeek 解析供應商/金額/日期
5. 建立 expense_receipt
6. Bot 回覆確認訊息

Bot 設定：`POST /api/messaging/telegram/setup` 接受 `bot_token`，自動註冊 webhook。

## 認證與密碼

- JWT (`jsonwebtoken`) — 24h token
- `authMiddleware` — 驗證 Bearer token
- 密碼 hash: `bcryptjs` (10 rounds)
- 新增員工：`POST /api/firms/:id/members` 支援 `password` 參數，回傳包含密碼

## 導航欄

`frontend/src/components/Layout.tsx`：
- 預設只顯示會計師常用功能（儀表板、文件處理、會計、會計師樓、設定）
- 側欄底部「顯示全部功能」checkbox 控制銷售/通訊/工具/客戶/帳戶/底部工具
- `hidden: true` 標記在 group 或 item 上
