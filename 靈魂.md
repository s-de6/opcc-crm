# 靈魂

## 一句話

讓香港會計師用 AI 管理客戶公司的文書、賬目與合規，從月結單 OCR 到記帳一條龍。AI 助理不只是查資料，還能寫程式碼、部署，像 Claude Code 內建在 CRM 裡。

## 為什麼要做

Mabel Lam（林家誼）是美國華盛頓州註冊執業會計師，在香港九龍紅磡經營會計師樓。她和同行每天的工作是：

- 收到客戶寄來的 HSBC 銀行月結單 PDF → 手動輸入交易
- 客戶 WhatsApp / Telegram 傳來帳單照片 → 手動分類記錄
- 每年追 BR 續期、CI、周年申報、利得稅 deadline → 靠 Excel / 記憶
- 年底把帳目 export CSV 給 auditor → 手動整理

這個系統把以上全部自動化。

## 核心設計原則

- **會計師視角優先** — 導航欄、功能順序按會計師工作流程設計，預設隱藏非核心功能（勾選「顯示全部功能」才出現）
- **文件驅動** — 上傳月結單 PDF → GLM-OCR 辨識 → DeepSeek 解析交易 → 自動分類科目，人只做確認
- **一間會計師樓管多間公司** — Firm 模式讓會計師切換客戶、管理權限；新增客戶時自動建立科目表與合規記錄
- **AI 無所不在** — 右側 AI 助理可以查資料、改程式碼、commit、部署。超出既有功能時不會說「做不到」，而是主動讀程式碼、寫新功能
- **AI 永不放棄** — 每次回應後自問「任務完成了嗎？」未完成就繼續分拆執行，不問用戶、不等待、不停止，直到任務徹底完成
- **香港會計實務** — 科目表 5 位數 4 層級、BR/CI/MPF/利得稅合規追蹤、財政年度設定、承上結餘
- **AI 記憶** — 導航欄有「AI 記憶」頁面，存放靈魂.md、技術記憶.md、賬本脈絡.md，AI 每次對話必須先讀取這三個檔案

## 使用者故事

1. **Mabel 早上登入** → 儀表板看到所有客戶的合規到期日、本月損益摘要
2. **客戶傳來 HSBC PDF** → 拖入文件上傳 → GLM-OCR 辨識 → DeepSeek 解析 35 筆交易 → 自動分類科目
3. **記帳頁** → 設定財政年度、承上結餘 → 從銀行資料自動產生分錄 → 查看試算表/損益表/資產負債表
4. **年底** → 導出 CSV 給 auditor
5. **要改功能** → 對 AI 說「把側欄顏色改藍色」→ AI 讀程式碼 → 修改 → commit → 部署
6. **客戶傳 Telegram 帳單照片** → Bot 接收 → GLM-OCR 辨識 → 自動存為消費單據 → 回覆確認訊息
7. **新增員工** → 在會計師樓管理頁面輸入 email + 密碼 → 員工即可登入

## 不做什麼

- 不做庫存管理（不是 ERP）
- 不取代 auditor（只做導出）
- 不做 payroll 計算（只記錄薪金支出科目）
- 不支援中國大陸稅制（只做香港）
